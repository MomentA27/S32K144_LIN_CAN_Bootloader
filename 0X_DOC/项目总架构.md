## 项目概述

NXP S32K144 的 LIN 引导加载程序（Bootloader），通过 LIN 总线 + UDS 协议实现固件升级（FOTA）。项目由 NXP Processor Expert 10.1 生成，在 S32 Design Studio (S32DS) 中开发。

- **目标 MCU**: NXP S32K144 (Cortex-M4F, 512KB P-Flash, 64KB SRAM)
- **工具链**: S32DS for ARM v2.x + GCC 6.3.1 arm-none-eabi
- **SDK**: S32_SDK_S32K1xx_RTM_3.0.0
- **通信**: LPUART1 作为 LIN 物理层，传输层遵循 ISO 17987-2
- **调试器**: PEMicro (SWD)，支持 Flash 和 RAM 两种调试配置

## 构建命令

本项目使用 S32DS IDE 管理构建，无独立的 Makefile/CMakeLists。构建方式：

```bash
# 在 S32DS IDE 中：
#   Project → Build Project (Ctrl+B) 编译
#   Run → Debug As → Debug_FLASH_PNE 下载并调试

# 或使用 EIDE (Embedded IDE) VS Code 扩展：
#   ctrl+shift+b → 选择 EIDE: Build 进行编译
#   按 F5 下载并调试
```

编译数据库位于 `build/Debug_FLASH/compile_commands.json`，可供 clangd 等 LSP 使用。

**构建配置**（定义于 `.cproject`）：
- 优化: C `-O1`, C++ `-O0`
- 标准: C99, C++17
- 浮点: hard abi + fpv4-sp-d16
- 库: newlib-nano + nosys
- 链接: `--gc-sections` (移除未用段)

**预定义宏**: `CPU_S32K144HFT0VLLT`, `CPU_S32K144`, `TURN_ON_CPU0`, `START_FROM_FLASH`

## 架构

```
main.c (入口 → BSP_init → 主循环)
  └─ UDS 应用层 (UDS_stack/UDS/) ← 尚未实现
       └─ TP 传输层 (UDS_stack/TP/)
            ├─ TP.c/h: 传输协议无关调度层
            └─ LIN_TP/: ISO 17987-2 LIN 传输层状态机
                 └─ FIFO (multi_cyc_fifo): 环形缓冲区队列
  ├─ Flash_app/: Flash 下载状态机 (擦除/编程/校验)
  ├─ HAL/: 硬件抽象层 (CRC/Flash/Timer/Watchdog/AES)
  ├─ auto_lib/: 最小化标准库 (fsl_memset/memcpy)
  └─ Debug/: 调试模块 (LPUART printf / GPIO 波形 / 计时器)

```

### 核心模块说明

**TP 传输层** (`Sources/unified_stack/UDS_stack/TP/`):
- `TP.h/TP.c` — 传输协议无关的上层接口，通过 `#ifdef EN_LIN_TP` / `EN_CAN_TP` 宏条件编译，委托到对应子模块
- `TP_cfg.h/TP_cfg.c` — TP 配置、FIFO 参数、回调类型定义，以及 TP-to-硬件驱动之间的数据交换
- `LIN_TP/LIN_tp.h/LIN_tp.c` — ISO 17987-2 LIN 传输层核心，约 1418 行状态机，8 个状态、表驱动分发。支持 SF(单帧)/FF(首帧)/CF(连续帧)，无流控帧(FC)
- `LIN_TP/LIN_tp_cfg.h/LIN_tp_cfg.c` — LIN TP 配置实例化（NAD 地址、N_As/N_Ar/N_Bs/N_Br/N_Cs/N_Cr 时序参数等），以及硬件驱动注入

**Flash_app** (`Sources/unified_stack/Flash_app/`):
- 下载四步骤: Request → Transfer → Exit → Checksum (UDS 0x34/0x36/0x37/0x31)
- Flash 操作通过 SRAM 中的函数指针执行（因为 Flash 编程/擦除时不能读 Flash）
- 支持 A/B 分区 FOTA（当前 `EN_SUPPORT_APP_B` 未启用）

**FIFO** (`Sources/unified_stack/FIFO/`):
- 多周期环形缓冲区，TP 层以此与硬件驱动交换数据
- 4 个 FIFO: RX_TP_QUEUE(150B), TX_TP_QUEUE(50B), RX_BUS_FIFO(50B), TX_BUS_FIFO(50B)
- 总缓冲 450 字节 (LIN 配置下)

**HAL 层** (`Sources/S32K144_LIN/HAL/`):
- flash_hal: Flash 操作函数表（擦除/编程/读取/校验）
- crc_hal: 硬件 CRC32 计算
- UDS_algorithm_hal: AES-128 软件实现（UDS 0x27 安全访问）
- timer_hal/watchdog_hal: 计时器/看门狗抽象

### 内存布局 (Flash)

| 区域 | 起始地址 | 大小 | 用途 |
|------|----------|------|------|
| m_interrupts | 0x00000000 | 1KB | 向量表 |
| m_flash_config | 0x00000400 | 16B | Flash 配置域 (FCF) |
| m_text | 0x00000410 | ~511KB | 代码+只读数据 |
| m_data (SRAM_L) | 0x1FFF8000 | 32KB | data段/code_ram/向量表副本 |
| m_data_2 (SRAM_U) | 0x20000000 | 28KB | BSS/堆/栈 |

启动流程: `Reset_Handler`(startup_S32K144.S) → `SystemInit()` → `init_data_bss()` → `main()` → `BSP_init()` → 主循环

## 配置

所有编译时配置集中在 `Sources/S32K144_LIN/public_inc/user_config.h`：

- `EN_LIN_TP` — 当前启用的传输协议（唯一）
- `RX_BOARD_ID=0x7F`, `RX_FUN_ID=0x7E`, `RX_PHY_ID=0x55`, `TX_ID=0x35` — LIN 诊断 ID
- `EN_CRC_HARDWARE` — 硬件 CRC
- `EN_ALG_SW` + `EN_ALG_HW` — 安全算法访问
- `EN_DEBUG_FLS_MODULE` — Flash 模块调试（可通过 `EN_TP_DEBUG` / `EN_UDS_DEBUG` 等开关开启其他模块调试）
- `DELAY_MAX_TIME_MS=5000` — 未收到 UDS 消息时，跳转 APP 前的等待时间

断言宏、调试打印宏在 `includes.h` 中定义，通过编译开关控制开启/屏蔽。

## 关键约定

- **单 TP 编译**: 必须且只能定义一个 TP 宏（`EN_LIN_TP` / `EN_CAN_TP` / `EN_ETHERNET_TP` / `EN_OTHERS_TP`），`includes.h` 中有编译期校验
- **中断安全**: Flash 写临界区通过 `DisableAllInterrupts()` / `EnableAllInterrupts()` 保护
- **Flash 操作**: 编程/擦除函数在执行前下载到 SRAM，通过函数指针调用（Flash 操作期间不能读 Flash）
- **看门狗**: 长擦除操作期间需喂狗，并通过 UDS 0x3E (TesterPresent) 向诊断仪请求更多时间
- **编码风格**: 注释和文档字符串均为中文；头文件使用 Doxygen 格式注释
- **命名**: 匈牙利命名法风格 — `gs_` (global static), `g_` (global), `pf` (pointer function), `e` (enum), `st` (struct), `uc/x` (unsigned char/int)
